# 安全说明

Poise 是个人自用的策略运行实验项目，源码公开不代表它已经适合托管他人的资金或生产账户。使用者需要自行承担配置、运行、交易、密钥和基础设施风险。

## API key

- 只给 Poise 使用独立 API key，不要复用主账户或其他系统的 key。
- 不要开启提现权限。
- 优先使用 testnet；连接 mainnet 前应先在 testnet 跑通完整启动、下单、暂停、恢复、平仓和重启流程。
- 不要把真实 `config.toml`、实例目录、SQLite 数据库、日志、`.env` 或任何密钥提交到仓库。
- 如果怀疑密钥泄露，立即在交易所后台禁用或删除该 key，再检查本地日志和提交历史。

## 运行风险

- Poise 可能自动提交、取消或修改合约订单。
- 交易所 API、WebSocket、网络、系统时钟、本地磁盘或进程崩溃都可能影响实际仓位。
- 策略参数、价格带、杠杆、损失限制和带外策略配置错误可能造成真实亏损。
- 本项目不保证收益、风控效果、数据完整性或持续可用性。

## 漏洞报告

这个仓库暂时不是社区型开源项目，也不承诺响应外部需求。若你发现会导致密钥泄露、未授权下单、错误连接 mainnet、绕过风险限制或破坏持久状态的问题，请通过仓库所有者可见的私密渠道报告，避免在公开 issue 中包含密钥、账户信息、订单信息或可直接复现攻击的细节。

如果没有私密渠道，请先公开提交一个不含敏感细节的最小提示，说明问题类别和影响范围，等待维护者确认后再提供复现细节。

# 网格示例区间总带宽收窄到 2000 美金设计

**日期：** 2026-03-28

**目标：** 将 Binance 测试网示例配置中的 `btc-core` 网格区间从过宽的静态范围收窄为总带宽 `2000 USD`，让本地联调时更容易观察到网格效果。

## 背景

当前示例配置文件 `configs/binance-testnet.toml` 中，`btc-core` 的价格区间是 `50000.0 -> 73000.0`，总带宽 `23000 USD`。这个区间对当前联调场景过宽，TUI 中只能看到很粗的范围描述，不利于验证网格带内行为。

## 决策

采用“围绕当前价格手动设置一个总带宽为 `2000 USD` 的示例区间”。

本次只调整示例配置，不引入自动按市价派生上下界的功能。

## 行为定义

### 1. 示例配置

- 只修改 `configs/binance-testnet.toml`
- `btc-core` 的 `upper_price - lower_price` 必须等于 `2000.0`
- 区间中心点按当前联调时的 BTC 价格手动选择

### 2. 示例文档

- 更新 `README.md` 中的测试网最小示例
- 文档中的示例区间与配置文件保持一致
- 补充说明“示例区间总带宽为 2000 美金，需要随市场变化手动调整”

## 架构落点

不改运行时逻辑，变更只落在：

- `configs/binance-testnet.toml`
- `README.md`
- `server/src/config.rs` 中的示例配置解析测试

## 非目标

- 不新增配置项
- 不自动读取实时价格生成区间
- 不修改 TUI 展示逻辑
- 不修改策略或撮合相关实现

## 测试策略

至少覆盖以下验收点：

1. `server/src/config.rs` 的示例配置解析测试断言 `btc-core` 区间总带宽为 `2000.0`
2. `README.md` 中的示例配置与实际配置文件一致

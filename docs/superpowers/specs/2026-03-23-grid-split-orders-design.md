# Grid 页双栏订单与距离显示设计

## 背景

当前 Grid 页左侧 `Strategy Orders` 只有一张表：

- 买单和卖单混在一起
- 用户需要先看 `Side` 再判断订单所处方向
- 右侧 `Grid Summary` 与 `Operator Notes` 采用单列堆叠，纵向空间占用偏高
- 订单到成交的远近只能靠人工比较价格，缺少直接可读的比例距离

这会让值守时的扫读成本偏高，尤其是在窄屏和多实例切换场景下。

## 目标

把 Grid 页改成更适合值守扫读的结构：

- `Strategy Orders` 按买单和卖单拆成左右两栏
- 每个订单显示相对当前最新价的比例距离
- `Grid Summary` 与 `Operator Notes` 改成双列 key-value 布局
- 保持当前服务端协议与选择器输入不变，只调整 TUI 选择器与渲染

## 非目标

- 不修改服务端数据结构
- 不改变“只展示当前真实策略挂单”的既有语义
- 不引入新的交互键位
- 不显示绝对价差，只显示比例距离

## 设计决策

### 1. 左侧订单区改成买卖双栏

`Strategy Orders` 区域拆成左右两个子表：

- 左栏：`BUY`
- 右栏：`SELL`

每栏只显示对应方向的真实策略挂单。这样用户进入页面后不需要先扫 `Side` 列，就能直接看出当前两侧挂单分布。

### 2. 距离只显示比例

每条订单新增“到成交距离”字段，定义为：

- `((order_price - last_price) / last_price) * 100`

显示格式使用百分比，例如：

- `-0.37%`
- `+0.29%`

不显示绝对价差，也不显示 `bps`。这样对多数用户更直观，不需要换算。

### 3. 同侧订单按更接近当前价优先

每个方向内部按距离当前最新价更近的顺序排列：

- 买单：价格更高的在前
- 卖单：价格更低的在前

这样每栏顶部就是最可能先成交的订单。

### 4. 右侧摘要改成双列 key-value

`Grid Summary` 与 `Operator Notes` 不再按一行一个字段纵向堆叠，而是改成两列成对展示，优先放天然成组的信息：

- `Status / Bias`
- `Lower / Upper`
- `Center / Span`
- `Active / Occupied`
- `Pending / Current Price`
- `Session / Health`
- `Breaker / 状态说明`

说明性文案仍保留在底部一行或两行，但主信息先以双列键值展示。

### 5. 空态保持现有语义

如果没有实时交易所挂单源，左右两栏仍保持空表，不回退到镜像单。  
本次只优化布局和信息表达，不改变订单来源规则。

## 验收标准

1. Grid 页左侧订单区拆成 `BUY` / `SELL` 两栏
2. 每条订单都显示相对当前最新价的百分比距离
3. 同一方向的订单按更接近当前价的顺序显示
4. `Grid Summary` 与 `Operator Notes` 改成双列 key-value 布局
5. 没有实时交易所挂单源时，Grid 页仍不显示镜像单

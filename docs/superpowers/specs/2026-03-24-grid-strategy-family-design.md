# 网格策略族模型设计

本文档定义一套新的网格策略族骨架。目标不是兼容当前实现，而是从零开始给出一份更干净的策略语义模型。

第一版只完整定义一个子策略：`永续合约震荡网格`。后续如果研究证明这条路线不对，可以整体推翻旧方案，不保留兼容负担。

## 1. 目标

- 用统一抽象定义网格策略，不把“挂一排单”当成策略本体
- 先把策略语义层讲清楚，再决定执行层如何落地
- 让 `中性 / 偏多 / 纯多 / 偏空 / 纯空` 落到同一个函数模型里
- 把策略边界限定在“固定价格带 + 容量预算 + 出区间处置”
- 把账户动态风险从策略本体剥离，交给独立风控程序

## 2. 非目标

本设计第一版明确不做：

- 不兼容当前固定区间梯子实现
- 不定义挂单、撤单、追价、maker / taker、拆单等执行细节
- 不在策略本体里处理账户权益、清算距离、组合保证金和多策略联动
- 不引入动态均衡价、动态移动价格带、自动重置区间
- 不把“价格网格层数”当成策略核心参数

## 3. 设计原则

### 3.1 策略本体是目标占用函数

这套模型把网格定义为：

- 一个固定价格带
- 一组多空容量预算
- 一个把价格映射到目标占用的函数
- 一个带外处置策略

它不把策略定义成一组静态挂单。

### 3.2 风险优先，但只管策略级预算

策略本体只回答：

- 在这个价格上，这个策略希望占用多少多头或空头容量

策略本体不回答：

- 当前账户还能承受多少风险
- 当前保证金是否足够
- 是否接近清算

这些由外层统一风控程序负责。

### 3.3 方向模式是常量配置，不是不同策略

`中性 / 偏多 / 纯多 / 偏空 / 纯空` 不再拆成五套策略逻辑，而是同一个目标占用函数配上不同的容量常量。

## 4. 核心对象

### 4.1 固定价格带

- `lower_price`
- `upper_price`

价格带定义策略的有效价格范围。

第一版约束：

- `lower_price < upper_price`
- 价格带创建后不自动漂移
- 不引入独立的“均衡价”或“锚点价”概念

价格带中点只作为一个价格参考点：

```text
band_center = (lower_price + upper_price) / 2
```

它不是零仓位点。只有在容量对称且函数形状对称时，中心点目标占用才会自然接近中性。

### 4.2 容量预算

- `long_capacity`
- `short_capacity`

这两个量表示该策略允许占用的最大多头容量和最大空头容量。

约束：

- `long_capacity >= 0`
- `short_capacity >= 0`
- 两者都属于策略自己的预算，而不是账户实时可用资产

语义说明：

- 容量不是价格层数
- 容量也不是账户总仓位上限
- 它表示这套策略在自己预算内最多能占用多少个多头或空头暴露单元

### 4.3 容量单元换算

容量单元到实际头寸的换算需要一个配置层参数：

- `capacity_notional`：每个容量单元对应的名义金额

`capacity_notional` 不属于策略语义核心，但第一版实现需要它来把抽象容量单位映射成合约数量。换算逻辑如下：

```text
capacity_unit_qty = capacity_notional / band_center
max_strategy_notional = max(long_capacity, short_capacity) * capacity_notional
```

约束：

- `capacity_notional > 0`
- 换算结果需经 `exchange_rules` 的精度和最小下单量约束对齐

更复杂的换算规则（按固定合约张数、按品种乘数等）属于后续扩展，不在第一版范围内。

## 5. 统一目标占用函数

### 5.1 标准表达

先把价格归一化到区间内位置：

```text
x = clamp((price - lower_price) / (upper_price - lower_price), 0, 1)
```

其中：

- `x = 0` 表示价格在下沿
- `x = 1` 表示价格在上沿

目标占用函数定义为：

```text
target_exposure(price) =
    -short_capacity + (long_capacity + short_capacity) * g(x)
```

其中 `g(x)` 满足：

- `g(0) = 1`
- `g(1) = 0`
- 在 `[0, 1]` 上单调递减

因此有：

- `price = lower_price` 时，`target_exposure = long_capacity`
- `price = upper_price` 时，`target_exposure = -short_capacity`

### 5.2 解释方式

为了避免把容量和价格层数混淆，推荐对外这样解释：

- `target_exposure > 0`：当前应占用若干多头容量
- `target_exposure < 0`：当前应占用若干空头容量
- `target_exposure = 0`：当前应回到中性占用

示例：

- `long_capacity = 8, short_capacity = 8`
  - 下沿：目标 `多头 8`
  - 中部：目标接近 `0`
  - 上沿：目标 `空头 8`
- `long_capacity = 8, short_capacity = 0`
  - 下沿：目标 `多头 8`
  - 中部：目标 `多头 4`
  - 上沿：目标 `0`

这自然表达了：

- 中性模式可以在区间两侧完整展开
- 纯多模式在对侧天然收敛到 `0`
- 纯空模式同理

## 6. 形状族

### 6.1 第一版约束

第一版不允许任意自定义曲线，只预留一小组受约束的 `shape_family`：

- `linear`
- `convex`
- `concave`

但第一版默认只以 `linear` 作为标准方案和验收基准。

### 6.2 具体公式

第一版三种形状族的 `g(x)` 统一定义为幂函数形式：

```text
g(x) = 1 - x^p
```

| shape_family | p   | 效果                                     |
|---|---|---|
| `linear`     | 1   | 目标占用随价格均匀变化                   |
| `convex`     | 2   | 更慢离开边缘容量，更晚向对侧切换         |
| `concave`    | 0.5 | 更快离开边缘容量，更早回到中性或低占用区 |

已知限制：第一版 `p` 值固定，不可调节。如果后续需要更细粒度的曲率控制，可以引入 `shape_param: f64` 参数化 `p`，但这不在第一版范围内。

### 6.3 不变量

无论选择哪种 `shape_family`，都必须满足：

- 只改变从 `long_capacity` 过渡到 `short_capacity` 的节奏
- 不改变方向模式定义
- 不改变价格带边界
- 不改变出区间语义
- 不改变容量预算本身

## 7. 方向模式

方向模式不再是独立策略类型，而是同一个函数配上不同容量常量。

### 7.1 中性

```text
long_capacity = N
short_capacity = N
```

语义：

- 多空两侧风险预算对称
- 在 `linear` 下，带中点附近自然接近中性占用

### 7.2 偏多

```text
long_capacity = N
short_capacity = M
0 < M < N
```

语义：

- 允许完整展开多头侧
- 允许较浅地展开空头侧
- 中心点目标占用天然偏多

### 7.3 纯多

```text
long_capacity = N
short_capacity = 0
```

语义：

- 只允许多头侧展开
- 上沿目标自然收敛到 `0`
- 不需要额外定义“对侧零仓位点”

### 7.4 偏空

```text
long_capacity = M
short_capacity = N
0 < M < N
```

语义与偏多对称。

### 7.5 纯空

```text
long_capacity = 0
short_capacity = N
```

语义与纯多对称。

## 8. 出区间处置

价格离开固定价格带后，策略不再继续按带内目标占用函数扩张风险。此时行为由 `out_of_band_policy` 定义。

### 8.1 `freeze`

语义：

- 带外冻结目标占用
- 不再扩大风险
- 也不主动收缩目标
- 价格回到带内后，自动恢复按带内函数计算

### 8.2 `reduce_only`

语义：

- 带外目标占用只允许单向收缩
- 只允许朝 `0` 或更安全方向移动
- 不允许带外继续增加风险

重要说明：

- `reduce_only` 只定义策略目标如何变化
- 不定义执行层用什么方式减仓
- 如果执行层暂时没有完成去风险，结果表现上可能暂时接近 `freeze`
- 但语义上它已经进入“应当去风险”的状态

### 8.3 `terminate`

语义：

- 带外进入退出流程
- 目标占用收敛到 `0`
- 当前这一轮策略生命周期结束

### 8.4 `hold`

语义：

- 带外保持当前目标占用不变
- 不要求自动恢复
- 更适合需要人工确认后再恢复的流程

与 `freeze` 的关系：`freeze` 和 `hold` 在目标占用计算上等价，都是冻结在离开带时的值。区别完全在操作层面：`freeze` 在价格回到带内后自动恢复，`hold` 需要人工或外部信号确认后才恢复。

### 8.5 共同约束

无论带外选择哪种策略，都必须满足：

- 带外不再增加风险
- 带外不再视为“正常运行中的震荡网格”
- 带外是策略有效性结束后的处置阶段

## 9. 与执行层的边界

本设计只定义策略想要的目标占用，不定义如何实现该目标。

### 9.1 策略层输出是理想值

`target_exposure` 是关于价格的连续函数，输出是理想值。执行层负责：

- 把理想值量化到交易所允许的离散价格和数量精度
- 在目标变化量小于阈值时不产生调仓动作（防抖 / 死区）
- 在带内中心区域，价格微幅抖动会让 `target_exposure` 在微小正负值之间来回变化，执行层必须用死区过滤这类无效信号

策略层不关心量化后的实际仓位是否与理想值精确相等。量化误差和防抖阈值完全属于执行层决策。

### 9.2 初始建仓语义

策略启动时，市场价在带内，`target_exposure` 可能是一个非零值。

第一版采用瞬间对齐语义：策略一旦进入 `Active` 状态，执行层应立即朝 `target_exposure` 调整仓位，不等待价格首次穿越。

这意味着启动时可能产生一笔较大的初始建仓单。执行层负责控制建仓节奏和滑点。

### 9.3 执行方式

执行层负责的问题包括：

- 如何把目标占用映射成真实订单
- 是否采用静态挂单、动态调仓、被动成交或混合执行
- 如何做拆单、撤单、价格偏移和节流

同一个策略模型可以被不同执行方式实现：

- 传统被动挂单
- 偏再平衡式调仓
- 被动挂单与主动收敛的混合执行

因此：

- "价格网格层数"如果未来需要恢复，也应该进入执行层，而不是策略本体
- 这套模型允许后续推翻执行方案，而不必推翻策略语义



## 10. 与风控层的边界

策略层只持有自己的容量预算，不动态消费账户级风险状态。

外层风控负责的问题包括：

- 账户权益变化
- 保证金安全
- 清算距离
- 多策略总风险
- 是否强制压制、暂停或终止某个策略

策略与风控的关系是：

- 策略给出“希望占用多少容量”
- 风控决定“当前是否允许策略继续占用”

这保证了：

- 策略语义稳定
- 风控职责集中
- 多策略共享一套账户级安全约束

## 11. 第一版默认建议

为了让第一版可解释、可验证、可被后续推翻，默认建议如下：

- 价格带固定，不自动移动
- 不引入独立均衡价
- 方向模式只通过 `long_capacity / short_capacity` 表达
- 默认 `shape_family = linear`
- 带外策略允许 `freeze / reduce_only / terminate / hold`
- 账户级动态风险完全放在策略外
- 初始建仓采用瞬间对齐语义

## 12. 不在本设计范围内但应尽快跟进的工作

以下工作项与本设计互补，不在策略语义层范围内，但应在本设计落地后独立推进：

- 参数护栏：单格毛利覆盖手续费、tick size 对齐、最小下单量等约束
- 合约风险语义：杠杆、清算距离、库存占用等合约特有信息的显式表达
- 收益拆分展示：grid profit / unrealized PnL / fees / inventory exposure 的独立统计

## 13. 后续探索方向

基于这套骨架，后续可以独立探索：

- 更合适的 `shape_family`，以及可调的 `shape_param`
- 价格带重置或漂移机制
- 执行层如何把目标占用高质量地翻译成订单
- 账户级风控如何压制或裁剪策略目标
- 多策略共享账户预算时的调度语义
- 同品种多实例的目标占用叠加与保证金计算

这些都不应反向污染第一版策略本体。
